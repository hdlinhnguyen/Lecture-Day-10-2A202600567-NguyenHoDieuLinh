1. Đăng ký tài liệu mới (SOP Quyền Truy Cập)
Contract
Tài liệu quy trình SOP access_control_sop chứa thông tin cấp quyền admin Level 4 bị cách ly do chưa được pipeline baseline đăng ký. Ta đã thêm nó vào Allowlist ở file Data Contract và cleaning rules.
# Khai báo trong transform/cleaning_rules.py
ALLOWED_DOC_IDS = frozenset({
    "policy_refund_v4", "sla_p1_2026",
    "it_helpdesk_faq", "hr_leave_policy",
    "access_control_sop"  # <-- Thêm đăng ký mới
})
2. Xung đột phiên bản chính sách nghỉ phép HR
Clean Rule
Dữ liệu xuất thô lẫn lộn giữa bản cũ 2025 (quy định 10 ngày phép) và bản mới 2026 (12 ngày phép). Logic kiểm tra sẽ cách ly các bản ghi cũ dựa vào ngày hiệu lực và từ khóa "10 ngày phép".
if doc_id == "hr_leave_policy":
    stale_kws = ["10 ngày phép năm", "10 ngày phép"]
    if any(kw in text for kw in stale_kws):
        quarantine.append({**raw, "reason": "stale_hr_policy_version"})
        continue
3. Làm sạch tiền tố rác OCR (Placeholder Prefix)
Clean Rule
Nhiều dòng dữ liệu bị dính tiền tố `"Nội dung không rõ ràng:"` hoặc `"!!!"`. Ta viết quy tắc làm sạch bằng cách cắt chuỗi và loại bỏ hoàn toàn các tiền tố bẩn này.
if fixed_text.startswith("Nội dung không rõ ràng:"):
    fixed_text = fixed_text[len("Nội dung không rõ ràng:"):].strip()
if fixed_text.startswith("!!!"):
    fixed_text = fixed_text.lstrip("!")
4. Làm giàu ngữ cảnh RAG (SLA P1 Context Enrichment)
RAG Retrieval Fix
Chunk P1 thô chỉ ghi `Escalation P1:` thiếu chữ `"Ticket"` khiến mô hình tìm kiếm ngữ nghĩa xếp hạng nó thấp hơn Ticket P2. Ta đã tự động làm giàu văn bản thành `Ticket P1 Escalation:` để kéo nó lên Top-1.
if doc_id == "sla_p1_2026":
    if fixed_text.startswith("Escalation P1:"):
        fixed_text = fixed_text.replace("Escalation P1:", "Ticket P1 Escalation:")
